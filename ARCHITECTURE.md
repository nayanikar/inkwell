# INKWELL — Architecture & Build Guide
> AI-native comic strip engine. You direct, agents draw.
> Built for SpacetimeDB Hackathon 2026 — Tracks: Best Use of LLMs, Best Web App

---

## What This Is

Inkwell is a collaborative storytelling engine that generates comic strip pages in real time. The user is a **director** — they set the genre, cast characters, and nudge the story. Three AI agents (character agents + a genre engine) collaborate inside SpacetimeDB to produce complete comic scenes, panel by panel.

Each **Scene** is a single comic page (5–7 panels) that tells one narrative beat. Scenes chain together into **Acts**. The user reads the scene, pokes it with directives, then advances to the next scene.

**The core loop:**
1. User fills setup form (genre, setting, characters)
2. Session initialised → SpacetimeDB tables populated
3. `generate_scene` procedure called → Anthropic writes dialogue + panel descriptions → OpenAI generates panel images
4. Comic page renders in real time as panel rows commit
5. User reads, optionally applies a nudge directive
6. User advances → next scene generates

## Tech Stack

| Layer | Choice | Why |
|---|---|---|
| Database + server | SpacetimeDB (Maincloud, free tier) | Subscriptions, reducers, procedures in one place. No separate backend. |
| Module language | TypeScript | `spacetime dev --template chat-react-ts` scaffold |
| Frontend | React + Vite + TypeScript | Auto-generated STDB bindings, fast dev loop |
| Styling | Tailwind CSS | Utility-first, fast in a hackathon |
| LLM (dialogue + scene) | Anthropic claude-sonnet-4-20250514 | Best narrative quality, structured JSON output |
| Image generation | OpenAI GPT Image 1 Mini (low quality) | ~$0.005/image, fast, enough for comic panels |
| Hosting (day 2) | Vercel (frontend) + STDB Maincloud | Free tier both |
| Fonts | Permanent Marker + Caveat + Special Elite | Comic aesthetic |

## SpacetimeDB Schema

All tables defined in `spacetimedb/src/index.ts`.

### Tables

```typescript
// Session — one row per story
Session {
  session_id:   u64       // PK, auto-increment
  genre:        string    // "noir" | "horror" | "comedy" | "fantasy" | "sci-fi" | "western" | "romance" | "thriller"
  setting:      string    // free text, e.g. "Rain-soaked 1950s Los Angeles"
  style_bible:  string    // locked style prompt injected into every image gen call
  total_scenes: u32       // how many scenes user requested (4 | 6 | 8 | 10)
  current_scene: u32      // which scene is active
  status:       string    // "setup" | "running" | "done"
  created_at:   u64
}
// Access: public: true

// Character — one row per character per session
Character {
  char_id:      u64       // PK
  session_id:   u64       // FK → Session
  name:         string
  archetype:    string    // "detective" | "femme fatale" | user-defined
  personality:  string    // 1-2 sentence description
  current_mood: string    // updated each scene by genre engine
  secret:       string    // private — only genre engine reads this
}
// Access: public: true (except secret field — handled in procedure logic)

// Memory — append-only event log per character
Memory {
  memory_id:    u64       // PK
  char_id:      u64       // FK → Character
  session_id:   u64       // FK → Session
  scene_num:    u32
  panel_num:    u32
  event_text:   string    // what this character witnessed/said/did
}
// Access: public: true

// NarrativeDirective — nudge rows broadcast to all agents
NarrativeDirective {
  directive_id: u64       // PK
  session_id:   u64       // FK → Session
  type:         string    // "plot" | "character" | "tone" | "tension" | "custom"
  content:      string    // the actual directive text
  applied_at_scene: u32
  applied_by:   string    // "user" | "genre_engine"
}
// Access: public: true

// Scene — one row per scene (comic page)
Scene {
  scene_id:     u64       // PK
  session_id:   u64       // FK → Session
  scene_num:    u32
  title:        string    // e.g. "The Rain Never Quits"
  status:       string    // "generating" | "done"
  created_at:   u64
}
// Access: public: true

// Panel — one row per panel within a scene
Panel {
  panel_id:     u64       // PK
  scene_id:     u64       // FK → Scene
  session_id:   u64       // FK → Session
  scene_num:    u32
  panel_num:    u32        // 1–7
  caption:      string    // narrator caption text
  speaker:      string    // character name (empty if no dialogue)
  dialogue:     string    // speech balloon text (empty if none)
  image_prompt: string    // prompt sent to OpenAI
  image_url:    string    // returned URL from OpenAI (empty while generating)
  layout_hint:  string    // "wide" | "tall" | "square" | "close-up" — for panel sizing
  status:       string    // "generating" | "done"
}
// Access: public: true
```

## SpacetimeDB Functions

### Reducers (state changes, no external HTTP)

```typescript
// Create a new session and seed characters
export const create_session = spacetimedb.reducer(
  { genre, setting, total_scenes, characters: CharacterInput[] },
  (ctx, args) => {
    const session_id = ctx.db.session.insert({ ...args, status: "setup" })
    args.characters.forEach(c => ctx.db.character.insert({ session_id, ...c }))
  }
)

// Apply a user nudge directive
export const apply_nudge = spacetimedb.reducer(
  { session_id, type, content },
  (ctx, args) => {
    const scene_num = ctx.db.session.findBySessionId(args.session_id).current_scene
    ctx.db.narrativeDirective.insert({ ...args, applied_at_scene: scene_num, applied_by: "user" })
  }
)

// Advance to next scene (called after user clicks "Next scene")
export const advance_scene = spacetimedb.reducer(
  { session_id },
  (ctx, { session_id }) => {
    const session = ctx.db.session.findBySessionId(session_id)
    ctx.db.session.updateBySessionId(session_id, {
      current_scene: session.current_scene + 1
    })
  }
)

// Update character mood (called by genre engine after each scene)
export const update_character_mood = spacetimedb.reducer(
  { char_id, new_mood },
  (ctx, args) => {
    ctx.db.character.updateByCharId(args.char_id, { current_mood: args.new_mood })
  }
)

// Append to character memory
export const append_memory = spacetimedb.reducer(
  { char_id, session_id, scene_num, panel_num, event_text },
  (ctx, args) => {
    ctx.db.memory.insert(args)
  }
)
```

### Procedures (can call external HTTP — Anthropic + OpenAI)

```typescript
// THE CORE PROCEDURE — generates a complete scene
// Called once per scene. Internally:
//   1. Reads character states + memory from DB (via withTx)
//   2. Reads any pending NarrativeDirectives
//   3. Calls Anthropic to generate scene structure (JSON)
//   4. Inserts Scene row + Panel rows with status="generating"
//   5. For each panel, calls OpenAI image API
//   6. Updates each Panel row with image_url + status="done"
export const generate_scene = spacetimedb.procedure(
  { session_id: t.u64(), scene_num: t.u32() },
  t.unit(),
  async (ctx, { session_id, scene_num }) => {

    // ── Step 1: Read context from DB ──
    const { session, characters, memories, directives } = ctx.withTx(tx => {
      const session = tx.db.session.findBySessionId(session_id)
      const characters = tx.db.character.filterBySessionId(session_id)
      const memories = tx.db.memory.filterBySessionId(session_id)
        .filter(m => m.scene_num >= scene_num - 2) // last 2 scenes of memory
      const directives = tx.db.narrativeDirective.filterBySessionId(session_id)
        .filter(d => d.applied_at_scene === scene_num)
      return { session, characters, memories, directives }
    })

    // ── Step 2: Call Anthropic for scene structure ──
    const sceneJson = await callAnthropic({
      session, characters, memories, directives, scene_num
    })
    // sceneJson shape: { title, panels: [{ caption, speaker, dialogue, image_prompt, layout_hint }] }

    // ── Step 3: Insert Scene + Panel rows (status=generating) ──
    let scene_id, panel_ids
    ctx.withTx(tx => {
      scene_id = tx.db.scene.insert({
        session_id, scene_num, title: sceneJson.title, status: "generating"
      })
      panel_ids = sceneJson.panels.map((p, i) =>
        tx.db.panel.insert({
          scene_id, session_id, scene_num,
          panel_num: i + 1,
          ...p,
          image_url: "",
          status: "generating"
        })
      )
    })
    // Client sees "generating" panels immediately — UI shows skeleton

    // ── Step 4: Generate images panel by panel ──
    for (let i = 0; i < sceneJson.panels.length; i++) {
      const panel = sceneJson.panels[i]
      const full_prompt = `${session.style_bible}. ${panel.image_prompt}`
      const image_url = await callOpenAI(full_prompt)

      ctx.withTx(tx => {
        tx.db.panel.updateByPanelId(panel_ids[i], { image_url, status: "done" })
      })
      // Each panel pops in on client as it completes
    }

    // ── Step 5: Mark scene done, update character moods ──
    ctx.withTx(tx => {
      tx.db.scene.updateBySceneId(scene_id, { status: "done" })
      sceneJson.character_updates?.forEach(u => {
        tx.db.character.updateByCharId(u.char_id, { current_mood: u.new_mood })
      })
      sceneJson.new_memories?.forEach(m => {
        tx.db.memory.insert({ session_id, scene_num, ...m })
      })
    })
  }
)
```

// lib/prompts.ts

export function buildScenePrompt({ session, characters, memories, directives, scene_num }) {
  return `
You are the genre engine for an AI comic strip engine called Inkwell.
You are the invisible author — you see everything, enforce genre logic, and ensure narrative coherence.

GENRE: ${session.genre}
SETTING: ${session.setting}
SCENE NUMBER: ${scene_num} of ${session.total_scenes}

CHARACTERS:
${characters.map(c => `- ${c.name} (${c.archetype}): ${c.personality}. Current mood: ${c.current_mood}`).join('\n')}

RECENT MEMORY (what characters have witnessed):
${memories.map(m => `- Scene ${m.scene_num}, Panel ${m.panel_num}: ${m.event_text}`).join('\n') || 'None yet — this is the opening scene.'}

ACTIVE DIRECTIVES (user nudges to honour):
${directives.map(d => `- [${d.type.toUpperCase()}] ${d.content}`).join('\n') || 'None — follow natural narrative progression.'}

YOUR TASK:
Generate a complete comic scene with 5–7 panels. This scene should read as one complete narrative beat — a beginning, middle, and end moment. Think Calvin & Hobbes strip: each panel advances, the last panel lands the emotional punch.

Respond ONLY with valid JSON. No preamble, no markdown fences. Schema:

{
  "title": "Short evocative scene title (4–6 words)",
  "panels": [
    {
      "panel_num": 1,
      "caption": "Narrator caption text (optional — empty string if none)",
      "speaker": "Character name or empty string",
      "dialogue": "Speech balloon text or empty string",
      "image_prompt": "Detailed visual description for image generation. Include: character descriptions, poses, expressions, environment, lighting, weather, camera angle (wide shot / close-up / medium shot). Style will be prepended separately.",
      "layout_hint": "wide | tall | square | close-up"
    }
  ],
  "character_updates": [
    { "char_id": 1, "new_mood": "updated mood string" }
  ],
  "new_memories": [
    { "char_id": 1, "panel_num": 3, "event_text": "Brief description of what this character witnessed" }
  ]
}

GENRE RULES for ${session.genre}:
${getGenreRules(session.genre)}
`.trim()
}

function getGenreRules(genre: string): string {
  const rules = {
    noir: "Someone always has a secret. Someone is always lying. Rain is always a character. Trust costs. The last line of dialogue should land like a verdict.",
    horror: "Build dread slowly. What is unseen is scarier than what is shown. The last panel should unsettle, not resolve.",
    comedy: "Timing is everything — the joke lands in the last panel. Characters should be earnest about absurd things.",
    fantasy: "Rules of magic must be consistent. Wonder is earned through specificity. The world reacts to character choices.",
    "sci-fi": "Technology is a mirror for character. The environment reflects the theme. Stakes are always larger than they appear.",
    western: "Silence carries as much weight as dialogue. Honour and pragmatism are always in tension. The landscape is a moral statement.",
    romance: "Misunderstanding drives tension. What is unsaid matters more than what is said. The last panel should be a held breath.",
    thriller: "Every panel should raise the stakes. Someone knows more than they are saying. Paranoia is the correct response.",
  }
  return rules[genre] || "Follow the internal logic of the world. Character consistency above all."
}

## OpenAI Image Call

```typescript
// lib/openai.ts

const STYLE_BIBLE_BASE = "Black and white comic strip art, hand-drawn ink style, expressive cartoon characters, clear linework, hatching for shadows, speech balloon compatible, newspaper comic aesthetic, no color"

export function buildStyleBible(genre: string, setting: string): string {
  return `${STYLE_BIBLE_BASE}. Setting: ${setting}. Tone: ${genre} genre.`
}

export async function callOpenAI(full_prompt: string): Promise<string> {
  const response = await fetch("https://api.openai.com/v1/images/generations", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "Authorization": `Bearer ${process.env.OPENAI_API_KEY}`
    },
    body: JSON.stringify({
      model: "gpt-image-1",          // GPT Image 1 Mini — cheapest
      prompt: full_prompt,
      n: 1,
      size: "1024x1024",
      quality: "low"                  // ~$0.005/image
    })
  })
  const data = await response.json()
  return data.data[0].url
}
```

## Client Subscription Pattern

```typescript
// lib/stdb.ts — how the React client wires up

import { DbConnection } from './module_bindings'

export const conn = DbConnection.builder()
  .withUri(import.meta.env.VITE_STDB_URI)
  .withModuleName("inkwell")
  .build()

// Subscribe to everything for this session
export function subscribeToSession(session_id: number) {
  conn.subscriptionBuilder()
    .onApplied(() => console.log("Subscription active"))
    .subscribe([
      `SELECT * FROM session WHERE session_id = ${session_id}`,
      `SELECT * FROM character WHERE session_id = ${session_id}`,
      `SELECT * FROM scene WHERE session_id = ${session_id}`,
      `SELECT * FROM panel WHERE session_id = ${session_id}`,
      `SELECT * FROM narrative_directive WHERE session_id = ${session_id}`,
      `SELECT * FROM memory WHERE session_id = ${session_id}`,
    ])
}

// React hook pattern
export function usePanels(session_id: number) {
  const [panels, setPanels] = useState<Panel[]>([])
  useEffect(() => {
    conn.db.panel.onInsert((panel) => {
      if (panel.session_id === session_id) {
        setPanels(prev => [...prev, panel])
      }
    })
    conn.db.panel.onUpdate((oldPanel, newPanel) => {
      setPanels(prev => prev.map(p => p.panel_id === newPanel.panel_id ? newPanel : p))
    })
  }, [session_id])
  return panels
}
```

## Screen Flow

```
SetupScreen
  → user fills: genre, setting, characters (2–4), scene count
  → calls: create_session reducer
  → calls: generate_scene procedure (scene 1)
  → navigates to: SceneScreen

SceneScreen (scene N)
  → subscribes to: panels for current scene
  → renders: ComicPage with panels popping in as image_url commits
  → shows: NudgeBar at bottom
  → on nudge: calls apply_nudge reducer
  → on "Next scene": calls advance_scene + generate_scene → new SceneScreen

SessionScreen (optional — accessible from header)
  → shows: all scenes generated so far as thumbnails
  → shows: character mood progression
  → shows: full story thread
```

## Design System

**Fonts:**
- Display / logo: `Permanent Marker` (Google Fonts)
- UI text + dialogue: `Caveat` (Google Fonts)  
- Labels / meta: `Special Elite` (Google Fonts)

**Colors:**
- Background: `#f5f0e8` (aged paper)
- Dark / ink: `#1a1612`
- Accent: `#c9510c` (comic red-orange)
- Gold: `#c9a96e` (noir amber)
- Surface: `#e8e0d0`

**Panel layout:**
- Row 1: 2–3 wide panels (establishing shots)
- Row 2: 3–4 small panels (action, close-ups)
- Row 3: 1–2 panels (scene closer / punch)
- Speech balloons: HTML overlay on image, not baked into image
- Captions: below panel, `Special Elite` font, paper background

## Key STDB Concepts for Cursor

1. **Reducers** = write to DB, no external HTTP, atomic. Use for all state mutations.
2. **Procedures** = can call external HTTP (Anthropic, OpenAI). Must use `ctx.withTx()` to read/write DB. The `withTx` callback may be called multiple times — never put `fetch()` calls inside `withTx`.
3. **Subscriptions** = client declares SQL queries it cares about. STDB pushes changes automatically. No polling needed.
4. **Auto-generated bindings** = run `spacetime dev` and `client/src/module_bindings/` is populated with typed TS classes matching your schema. Use these, don't write raw SQL on the client.
5. **Procedures are beta** = stable enough for this use case. Add `// @ts-ignore` if TS complains about procedure types.
